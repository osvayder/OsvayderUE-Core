# Roadmap

## Phase 1: Public Beta Hardening

- Keep the verified baseline explicit: Unreal Engine 5.7 on Windows.
- Evaluate additional Unreal Engine versions before widening compatibility claims.
- Add Linux and macOS validation notes only after clean-host checks pass.
- Finish license and attribution review for bundled subcomponents.
- Replace placeholder host-project notes with a tiny generated-state-free example when practical.

## Phase 2: Reviewer and Demo Experience

- Keep installation, verification, demo, and backlog docs aligned with the public repository layout.
- Refresh screenshots when the visible editor UI changes.
- Add a short demo video only after a public-safe capture exists.
- Document known caveats near the install path instead of burying them in internal proof notes.

## Phase 3: OpenAI and Codex Maintainer Workflows

- Maintain a clear Codex-oriented contributor path for issue triage, local reproduction, and regression checks.
- Document MCP bridge setup, auth assumptions, and non-goals for public maintainers.
- Add automated checks for docs hygiene, private-path leaks, and stale compatibility claims.
- Improve reviewer handoff docs without exposing local machine paths or internal operational labels.

## Phase 4: Community Readiness

- Convert the public backlog into GitHub issues after repository permissions are available.
- Add contribution guidance for bug reports, minimal repro projects, and screenshot evidence.
- Label beginner-friendly docs and validation tasks.
- Track platform compatibility and Unreal version support in public release notes.
